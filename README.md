# LINUX DO 阅读助手使用说明

## 作用

这是一个用于 LINUX DO 的篡改猴脚本。它会在页面右下角显示控制面板，按你设置的节奏打开话题、滚动阅读帖子，并在读完后返回列表继续。

脚本只在你手动点击“开始”后运行。

## 安装

1. 打开 Chrome 的篡改猴。
2. 新建脚本。
3. 将 `scripts/linux-do-reader/linux-do-reader.to-paste.user.js` 的内容完整粘贴进去。
4. 保存脚本。
5. 打开或刷新 `https://linux.do/latest`。

刷新后页面右下角会出现“LINUX DO 阅读助手”面板。

## 基本操作

- `开始`：按当前设置开始阅读。
- `停止`：停止当前运行。
- `清空已读记录`：清空“帖子优先”模式下记录过的热门话题 id。
- `重置设置`：恢复面板参数为默认值，不清空已读记录。

## 推荐默认设置

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| 优先策略 | 话题优先 | 默认从未读/新话题中选择 |
| 主帖点赞 | 关闭 | 开启后会给主帖点爱心 |
| 话题上限 | 10 | 每轮最多处理的话题数 |
| 每话题最多帖子 | -1 | `-1` 表示读完整个话题 |
| 主帖停留(ms) | 2500 | 进入话题后先停留主帖 |
| 最短停留(ms) | 1200 | 每次滚动后的最短停留 |
| 最长停留(ms) | 2000 | 每次滚动后的最长停留 |
| 冷却最短(ms) | 3000 | 话题结束后的最短冷却 |
| 冷却最长(ms) | 8000 | 话题结束后的最长冷却 |

## 优先策略

### 话题优先

适合提高已读话题数。

逻辑：

1. 从最新/未读话题列表选择带小蓝点或未读状态的话题。
2. 优先选择回复较少的话题。
3. 进入话题后从主帖开始阅读。
4. 读完后冷却，再返回主页继续。

### 帖子优先

适合阅读回复数量多的话题。

逻辑：

1. 自动打开 `https://linux.do/hot`。
2. 从热门话题中选择回复数量最多的话题。
3. 热门话题即使没有小蓝点也会被选择。
4. 每读完一个热门话题，记录它的 topic id。
5. 后续会跳过已经记录过的热门话题。

如果想重新从热门话题开始，可以点击 `清空已读记录`。

## 每话题最多帖子

这个参数用于控制每个话题最多读到多少帖子。

- `-1`：不限制，读完整个话题。
- `3`：读到 `3 / 总数` 后结束当前话题。
- `20`：读到 `20 / 总数` 后结束当前话题。

这个功能主要用于减少单个超长话题的阅读时间和页面加载压力。

## 主帖点赞

`主帖点赞` 默认关闭。

开启后，脚本会在进入话题并等待主帖停留后，尝试点击主帖的爱心按钮。

脚本只会点击标题为“点赞此帖子”的按钮。如果主帖已经点过赞，按钮通常会变成“移除此赞”或类似文案，脚本不会再点击，避免取消点赞。

## 状态说明

面板底部会显示当前状态，例如：

- `打开话题 1/10，策略：话题优先`
- `阅读中 1/10 6 / 35 未读标记:2`
- `达到每话题 20 帖上限，准备返回主页`
- `话题已读完，冷却 5 秒后返回主页`
- `完成：已读 10 个话题`

## 调试建议

首次测试建议这样设置：

| 参数 | 建议值 |
| --- | ---: |
| 话题上限 | 1 |
| 每话题最多帖子 | 3 |
| 主帖点赞 | 关闭 |
| 冷却最短(ms) | 1000 |
| 冷却最长(ms) | 1500 |

确认流程正常后，再恢复默认或调大参数。

## 注意事项

- 脚本依赖 LINUX DO 当前页面 DOM，站点改版后可能需要调整选择器。
- 不建议长时间无人值守运行。
- `重置设置` 不会清空热门已读记录。
- `清空已读记录` 只影响“帖子优先”的热门话题跳过逻辑。
- 如果页面卡住，先点击 `停止`，再刷新页面。
