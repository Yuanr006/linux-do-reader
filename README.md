# LINUX DO 阅读助手

## 简介

这是一个用于 LINUX DO 的篡改猴脚本。脚本会在页面右下角显示控制面板，按设定节奏自动打开话题、滚动阅读帖子，并在读完后返回列表继续。

脚本只会在手动点击 `开始` 后运行。

## 功能说明

- 支持 `话题优先`：从最新/未读列表中选择带小蓝点或未读状态的话题，优先阅读回复较少的话题。
- 支持 `帖子优先`：自动进入 `https://linux.do/hot`，优先阅读回复数量较多的热门话题；再次进入同一话题时依赖网站自身阅读进度继续。
- 支持设置每轮话题上限、每话题最多帖子数、滚动停留时间和话题冷却时间。
- 支持可选主帖点赞，只会点击标题为“点赞此帖子”的按钮，避免取消已有点赞。
- 每次滚动后会统计当前安全视口内的小蓝点；达到未读等待阈值后，会等小蓝点消失再继续滚动。
- 贴近屏幕顶部或底部边缘的小蓝点不会阻塞滚动，避免长话题中边缘标记卡住。
- 检测到推荐话题区域后会结束当前话题，不继续向下滚动。
- 滚动节奏默认较慢，并在话题之间加入冷却时间，减少请求和滚动频率。
- 如果小蓝点短时间不消失，脚本会二次复查；确认仍未消失后会小幅继续下滑再复查，避免站点已读状态延迟导致停止。
- 如果连续多次复查后小蓝点仍不消失，脚本会停止并弹窗提示：换 VPN 节点，访问 `https://linux.do/challenge`（提示“页面不存在或者是一个不公开页面”是正常的），稍后再继续。
- 面板会显示当前脚本版本和运行状态，例如阅读进度和未读标记数量；只有正在等待小蓝点消失时才显示等待数量。
- 启动后面板会自动切换为紧凑状态，只保留运行状态和停止按钮。

## 安装部署使用

1. 打开 Chrome 的篡改猴。
2. 新建脚本。
3. 将 `scripts/linux-do-reader/linux-do-reader.to-paste.user.js` 的内容完整粘贴进去。
4. 保存脚本。
5. 打开或刷新 `https://linux.do/latest`。
6. 页面右下角出现 `LINUX DO 阅读助手` 面板后，按需调整参数。
7. 点击 `开始` 运行，点击 `停止` 可随时停止。

面板操作：

- `开始`：按当前设置开始阅读。
- `停止`：停止当前运行。
- `重置设置`：恢复面板参数为默认值。

常用参数：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| 优先策略 | 话题优先 | 从未读话题或热门话题中选择阅读目标 |
| 主帖点赞 | 关闭 | 是否尝试给主帖点赞 |
| 话题上限 | 10 | 每轮最多处理的话题数 |
| 每话题最多帖子 | -1 | `-1` 表示不限制，读完整个话题 |
| 未读等待阈值 | 3 | 安全视口内未读帖子数达到该值后，等待小蓝点清零再继续滚动 |
| 最短停留(ms) | 1500 | 每次滚动后的最短停留时间 |
| 最长停留(ms) | 5200 | 每次滚动后的最长停留时间 |
| 冷却最短(ms) | 3000 | 话题结束后的最短冷却时间 |
| 冷却最长(ms) | 8000 | 话题结束后的最长冷却时间 |

## 注意事项

- 脚本依赖 LINUX DO 当前页面 DOM，站点改版后可能需要调整选择器。
- 不建议长时间无人值守运行。
- 如果页面卡住，先点击 `停止`，再刷新页面。
- 如果弹出小蓝点持续不消失提示，按弹窗提示处理后再继续。
